# 스킬: Tool Reliability 강화

## 목적
- 툴 호출 실패율을 낮추고, 재시도/폴백 전략으로 안정성을 높입니다.

## 입력
- 사용 중인 도구 목록
- 실패 로그/에러 패턴

## 출력
- 도구별 신뢰성 개선 계획
- 재시도/폴백 규칙 및 로깅 기준

## 절차
1. **실패 패턴 분류**: 네트워크, 인증, 파라미터 오류 등 유형화.
2. **재시도 정책 수립**: 백오프, 최대 횟수, 복구 기준 정의.
3. **폴백 경로 설정**: 대체 도구 또는 안전한 기본값 지정.
4. **리플레이/Mock 전략**: 회귀 테스트용 재현 환경 구축.
