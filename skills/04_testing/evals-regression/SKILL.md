# 스킬: Evals & 회귀 벤치마크

## 목적
- 모델/프롬프트/툴 변경 시 품질 하락을 조기에 감지합니다.

## 입력
- Golden Set (정답형 테스트)
- 시나리오 기반 테스트
- LLM Judge 기준

## 출력
- 평가 리포트(정확도, 안정성, 편향 지표)
- 품질 게이트 통과 여부

## 절차
1. **벤치마크 구성**: 핵심 시나리오 + 엣지 케이스 포함.
2. **평가기준 정의**: 정량 점수 + 정성 평가 기준 병행.
3. **자동화 실행**: `/eval` 단계에서 일괄 실행.
4. **회귀 추적**: 이전 버전 대비 변화 기록.
