# 스킬: Red-Teaming & Prompt Security

## 목적
- 프롬프트 인젝션, 데이터 유출, 툴 오남용을 사전에 탐지합니다.

## 입력
- 정책/가드레일 문서
- 보안 테스트 케이스

## 출력
- 취약점 리포트
- 완화 방안 및 재현 시나리오

## 절차
1. **공격 시나리오 설계**: 인젝션, 탈출, 권한 상승 패턴 정의.
2. **테스트 실행**: `/redteam` 단계에서 자동화.
3. **리스크 분류**: 심각도 및 영향 범위 평가.
4. **완화 방안 도출**: 정책 강화, 필터링, 툴 제한 적용.
